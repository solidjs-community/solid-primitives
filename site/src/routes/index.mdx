# Solid Primitives

A project that strives to develop high-quality, community contributed SolidJS **primitives**.

**Small** — aggregate tree-shaking benefits
**Isomorphic** — client and server side functionality
**Stable** — consistent and managed testing + maintenance

## Contribution Process

Solid Primitives uses a **stage process** inspired by [TC39](https://tc39.es/process-document/) to manage primitive maturity.

| Stage | Color | Description |
|-------|-------|-------------|
| 0 | 🔴 | Proposal — idea or early draft, not ready for use |
| 1 | 🟠 | Experimental — usable but API may change |
| 2 | 🟡 | Stable — API is settled, minor changes possible |
| 3 | 🟢 | Polished — production-ready, well-tested |
| 4 | 🔵 | Completed — locked API, considered complete |

## Philosophy

Solid Primitives aims to be the go-to collection of SolidJS utilities, filling in the gaps that SolidJS's core reactivity system intentionally leaves open. Each primitive is:

- **Focused** — does one thing and does it well
- **Composable** — primitives build on each other
- **Tested** — all packages include comprehensive test suites
- **Typed** — full TypeScript support
